# Watch Tower - 1차 개발 로드맵

## 1. 1차 개발 원칙

- 신규 모델 개발이 아니라 `공개 모델 선정 + 최소 튜닝 + 현장 검증`으로 진행
- 서비스 개발보다 `핵심 기능 모델 검증`을 우선
- 혼자 작업 기준으로 범위를 최소화

## 2. 1차 개발/검증 대상

- `Person Detection`
- `Tracking`
- `위험구역 침입 감지`
- `구역 내 인원 탐지`
- `PPE 감지`

## 3. 1차 기능 구조

- `Person Detection` : 공통 기반 모델
- `Tracking` : 공통 기반 알고리즘
- `위험구역 침입 감지` : `Person Detection + Tracking + Zone Rule`
- `구역 내 인원 탐지` : `Person Detection + Tracking + Counting Rule`
- `PPE 감지` : 별도 공개 모델 사용, 최소 튜닝 적용

## 4. 1차 우선순위

1. `Person Detection`
2. `Tracking`
3. `위험구역 침입 감지`
4. `구역 내 인원 탐지`
5. `PPE 감지`

## 5. 1차 제외 범위

- `쓰러짐 감지`
- `화재 감지`
- `출입통제`
- `정식 서비스 개발`
- `복잡한 대시보드`

## 6. 1차 작업 순서

1. 현장 영상 수집
2. `Person Detection` 공개 모델 선정
3. `Person Detection` baseline 검증
4. `Tracking` 적용 및 검증
5. `Zone Rule` 적용
6. `위험구역 침입 감지` 검증
7. `구역 내 인원 탐지` 검증
8. `PPE` 공개 모델 선정 및 검증
9. 최소 튜닝
10. 실패 케이스 정리 및 Go/No-Go 판단

## 7. 1차 산출물

- 기능별 검증 결과
- 기능별 실패 케이스 목록
- 최종 사용 모델/알고리즘 선정안
- 2차 개발 대상 확정

## 8. 핵심 결론

- 1차는 `사람 검출`부터 시작
- `위험구역 침입`과 `구역 내 인원 탐지`는 별도 모델이 아니라 규칙 기반 기능으로 검증
- `PPE`는 별도 공개 모델을 가져와 최소 튜닝 후 검증
- 1차 목표는 `핵심 기능 성립 여부 확인`
