# Watch Tower - 1차 개발 에코시스템

## 1. 목적

1차 개발 단계에서 `핵심 기능 모델 검증`을 빠르고 단순하게 진행하기 위한 최소 에코시스템을 정리한다.

## 2. 최소 권장 도구

- `CVAT`
- `supervision`
- `FiftyOne`

## 3. 도구별 역할

### 3.1 CVAT

**역할:** 데이터 라벨링

**주요 사용 대상**
- `Person Detection`
- `PPE 감지`
- 비디오 기반 bbox 라벨링
- keyframe 기반 라벨링

**활용 방식**
- 현장 영상에서 사람 bbox 라벨링
- PPE 클래스 라벨링
- 비디오 keyframe 지정
- interpolation으로 중간 프레임 자동 보간
- tracking 기반 객체 라벨링

**선정 이유**
- 비디오 라벨링에 적합
- keyframe / interpolation 지원
- tracking 작업 지원
- 1차 검증용 데이터셋 구축에 가장 직접적

### 3.2 supervision

**역할:** 위험구역 침입, 구역 내 인원 탐지, 라인 카운팅 구현 보조

**주요 사용 대상**
- `위험구역 침입 감지`
- `구역 내 인원 탐지`
- `입장/퇴장 카운팅`

**활용 방식**
- `PolygonZone`으로 위험구역 판정
- `LineZone`으로 입장/퇴장 카운팅
- `ByteTrack` 연동
- bbox 시각화
- zone/counting 로직 빠른 검증

**선정 이유**
- 1차 핵심 기능과 직접 연결됨
- 직접 구현해야 할 로직을 줄여줌
- 사람 검출 결과를 바로 기능 검증으로 연결 가능

### 3.3 FiftyOne

**역할:** 예측 결과 검토, 실패 케이스 분석, 평가

**주요 사용 대상**
- `Person Detection` 결과 검토
- `PPE 감지` 실패 케이스 분석
- 기능별 오탐/미탐 분석

**활용 방식**
- GT와 예측 결과 비교
- 오탐/미탐 샘플 필터링
- 실패 케이스 탐색
- 데이터셋 품질 점검
- 기능별 검증 결과 시각화

**선정 이유**
- 숫자 외에 실패 원인을 빠르게 파악 가능
- 혼자 작업할 때 결과 검토 효율이 높음
- 1차 Go / No-Go 판단에 직접 도움

## 4. 1차 작업 흐름에서의 사용 방식

1. `CVAT`으로 현장 영상 라벨링
2. 공개 모델 추론 실행
3. `supervision`으로 위험구역 침입 / 구역 내 인원 탐지 검증
4. `FiftyOne`으로 결과 검토 및 실패 케이스 분석

## 5. 현재 단계에서 제외하는 도구

- `Weights & Biases`
- `MLflow`
- `Kubeflow`
- `Airflow`
- 복잡한 MLOps 플랫폼
- 대형 자동화 플랫폼

## 6. 핵심 결론

- 1차에서는 과한 MLOps보다 `라벨링`, `기능 검증`, `실패 분석` 도구가 더 중요하다
- 혼자 작업 기준 최소 에코시스템은 `CVAT + supervision + FiftyOne`이면 충분하다
