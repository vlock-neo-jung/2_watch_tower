# Step 3: Event-level GT(Ground Truth) 준비

## 이 단계가 왜 필요한가

Phase 2에서 Person Detection 정확도를 측정할 때는 SODA 데이터셋의 GT(정답 라벨)가 있었다.
"이 이미지에서 사람이 어디에 있는가"가 이미 표시되어 있으니까, 모델의 예측과 비교하면 됐다.

Zone 침입 감지도 정확도를 측정하려면 **정답이 필요**하다.
"이 영상의 몇 초~몇 초 구간에서 이 구역에 침입이 있었다"라는 정답 데이터.
문제는 **이런 데이터셋이 공개적으로 존재하지 않는다는 것**이다.

따라서 우리가 직접 만들어야 한다. 단, 대규모로 만들 필요는 없다.

## 왜 frame-level이 아니라 event-level인가

두 가지 평가 방식의 차이:

### Frame-level 평가
```
프레임 1: 침입 있음(GT) → 시스템 감지함 → TP
프레임 2: 침입 있음(GT) → 시스템 놓침 → FN
프레임 3: 침입 있음(GT) → 시스템 감지함 → TP
프레임 4: 침입 없음(GT) → 시스템 감지함 → FP
...
```

**문제**: 하나의 침입 사건(30프레임 동안 사람이 zone 안에 있음)에서 25프레임은 감지하고 5프레임은 놓쳤다면?
frame-level로는 Recall 83%로 나쁘지 않아 보인다.
하지만 실제로는 **침입을 감지한 것**이다. 안전 관리자 입장에서 이 침입은 "감지 성공"이다.

반대로, 10건의 침입 중 9건은 완벽히 감지했지만 1건을 완전히 놓쳤다면?
frame-level Recall은 높게 나올 수 있지만, **실제로 1건의 위험을 놓친 것**이다.

### Event-level 평가 (i-LIDS 방식)
```
침입 사건 1 (프레임 100~200): 시스템이 프레임 105에서 감지 → TP (5프레임 = 1초 만에 감지)
침입 사건 2 (프레임 500~600): 시스템이 전혀 감지 못함 → FN
침입 사건 3 (프레임 800~900): 시스템이 프레임 810에서 감지 → TP
프레임 1200에서 시스템이 오탐 발생 → FP
```
→ Event Detection Rate: 2/3 = 67%, False Alarm: 1건

**이것이 안전 시스템의 실제 성능을 더 정확하게 반영한다.**

영국 정부의 i-LIDS 벤치마크가 이 방식을 사용한다:
> "침입 시작 후 10초 이내에 최소 1개 알람이 발생하면 True Positive"

## 구체적으로 무엇을 만드는가

### 1. 테스트 영상 선정

이미 보유한 샘플 영상 중 사람이 이동하는 장면이 있는 영상 1개를 선정한다.
이상적으로는:
- 사람이 특정 영역을 들어갔다 나오는 장면
- 여러 명이 다른 시점에 비슷한 영역에 출입하는 장면
- 최소 1분 이상 길이

### 2. 테스트용 zone 정의

선정한 영상의 스냅샷을 보고, 사람이 드나드는 영역에 polygon zone 1~2개를 정의한다.
이 zone은 실제 위험구역이 아니라 **측정 목적으로 임의로 설정**하는 것이다.
핵심은 "사람이 들어갔다 나오는 이벤트가 여러 번 발생하는 위치"에 zone을 그리는 것.

### 3. 수동 어노테이션

영상을 직접 보면서 "이 사람이 zone에 들어간 프레임"과 "나간 프레임"을 기록한다.

```
영상: construction_subway_cctv.mp4 (FPS: 25)
Zone: test_zone_01

  침입 1: 프레임 150~300 (6초간)  — 작업자 A가 zone 통과
  침입 2: 프레임 520~580 (2.4초)  — 작업자 B가 zone 횡단
  침입 3: 프레임 820~950 (5.2초)  — 작업자 C가 zone 내 작업
  ...
```

**소요 시간**: 1분 영상 기준 15~30분 정도.
정확한 프레임 번호는 대략적이어도 된다(±5프레임 허용).
event-level 평가이므로 프레임 단위 정밀함은 필요 없다.

### 4. GT 파일 포맷

```json
{
  "video": "construction_subway_cctv.mp4",
  "video_fps": 25,
  "zones": [
    {
      "zone_id": "test_zone_01",
      "geometry": [[0.15, 0.30], [0.60, 0.25], [0.65, 0.85], [0.20, 0.90]],
      "events": [
        {
          "type": "intrusion",
          "start_frame": 150,
          "end_frame": 300,
          "description": "작업자가 zone 통과"
        },
        {
          "type": "intrusion",
          "start_frame": 520,
          "end_frame": 580,
          "description": "작업자가 zone 횡단"
        },
        {
          "type": "intrusion",
          "start_frame": 820,
          "end_frame": 950,
          "description": "작업자가 zone 내 작업"
        }
      ]
    }
  ]
}
```

### 어노테이션 도구 사용

별도로 설계한 **Zone 설정 + GT 어노테이션 도구** ([03-1-annotation-tool-spec.md](./03-1-annotation-tool-spec.md))를 사용한다.

작업 흐름:
1. 도구에서 영상을 열고, **Zone 편집 모드**로 zone polygon을 그린다 → zone 설정 YAML 저장
2. **어노테이션 모드**로 전환하여 영상을 재생하면서 침입 구간을 마킹한다 → GT JSON 저장

도구가 Zone 그리기와 GT 어노테이션을 모두 처리하므로,
Step 2(zone 설정 스키마)의 샘플 데이터도 이 도구로 생성할 수 있다.

### 최소 규모로 시작

영상 1개, zone 1~2개, 이벤트 5~10개면 첫 평가에 충분하다.
결과가 유의미하면 더 추가한다.
