# Step 6: Go/No-Go 판단

## 이 단계가 왜 필요한가

Phase 2에서도 Person Detection 벤치마크 결과를 보고 "이 수준이면 현장에서 쓸 수 있는가?"를 판단했다.
Zone 침입 감지도 마찬가지다. Step 5의 수치를 보고 **다음 중 하나를 결정**해야 한다:

- **Go**: Phase 3 본격 구현 진입 (zone 설정 UI, 이벤트 시스템 등)
- **Conditional**: 특정 문제를 해결하면 진행 가능
- **No-Go**: 근본적 접근 방식 재검토 필요

## 판단 기준

### 필수 조건 (모두 충족해야 Go)

| 메트릭 | Go | Conditional | No-Go |
|--------|-----|-------------|-------|
| Event Detection Rate | ≥ 80% | 60-80% | < 60% |
| False Alarm Rate | ≤ 2건/zone | 3-5건/zone | > 5건/zone |
| Response Time | ≤ 2초 (@5fps) | 2-5초 | > 5초 |

### 왜 이 수치인가

- **EDR ≥ 80%**: 10건의 침입 중 8건 이상 감지. 안전 시스템에서 2건을 놓치는 것도 위험하지만,
  현재는 모델 검증 단계이므로 80%를 1차 목표로 잡는다. 상용화 시에는 90%+ 필요.

- **False Alarm ≤ 2건/zone**: 오탐이 많으면 관리자가 알림을 무시하게 된다.
  "양치기 소년 효과"를 방지하려면 오탐이 극히 적어야 한다.
  테스트 영상 1개 기준 zone당 2건 이하를 목표로 한다.

- **Response Time ≤ 2초**: 사람이 위험구역에 들어온 후 2초 이내에 감지해야
  경고음/알림이 의미가 있다. 5fps에서 10프레임 = 2초.

## Conditional일 때 원인 분석 방향

### 원인 1: Detection 문제 (Layer 1)

**증상**: 사람이 zone 안에 있지만 감지 자체가 안 됨 → FN 증가
**진단 방법**: 결과 영상에서 미감지 프레임 확인
**해결 방향**:
- 파인튜닝 모델로 교체 후 재측정
- SAHI 적용으로 소형 객체 감지 개선
- 신뢰도 임계값(conf) 낮추기 (단, FP 증가 가능)

### 원인 2: Tracking 문제 (Layer 2)

**증상**: 같은 사람의 ID가 바뀌면서 연속 프레임 카운트가 리셋됨 → 이벤트 발생 지연
**진단 방법**: 결과 영상에서 track ID 변화 확인
**해결 방향**:
- ByteTrack 파라미터 조정 (track_buffer 늘리기)
- BoT-SORT로 전환 (ReID 기능으로 재식별 개선)

### 원인 3: Zone Logic 문제 (Layer 3)

**증상**: 사람이 zone 안인데 밖으로 판정, 또는 반대
**진단 방법**: 경계선 근처 프레임에서 판정 결과 확인
**해결 방향**:
- anchor point 변경 (BOTTOM_CENTER → CENTER 등)
- polygon을 약간 확장하여 마진 확보
- 다중 anchor 사용 (2개 anchor 중 하나만 안에 있어도 True)

### 원인 4: Temporal Logic 문제 (Layer 4)

**증상**: 이벤트 발생이 너무 느리거나, 오탐이 너무 많거나
**진단 방법**: min_consecutive_frames 값과 결과 비교
**해결 방향**:
- min_consecutive_frames 조정 (3 ↔ 5 ↔ 10)
- cooldown_ms 조정
- 이 값은 실험적으로 최적값을 찾아야 함

## 판단 후 다음 행동

### Go인 경우

Phase 3 로드맵(4/5~4/18)에 따라 본격 구현:
1. zone 설정 관리 시스템 (CRUD, 저장, 이력)
2. 이벤트 생성 및 저장
3. 영상 기반 통합 테스트
4. 인원 카운팅 기능 추가 (같은 PolygonZone 기반)

### Conditional인 경우

원인별 해결 작업 → 재측정 → 재판단.
파인튜닝 모델 도착이 해결의 핵심이라면, 모델 도착 후 재측정만으로 Go 전환 가능.

### No-Go인 경우

근본적 문제 분석:
- 카메라 각도/해상도가 zone 판정에 부적합한가?
- 사람 감지 자체가 건설 현장 조건에서 불안정한가?
- 다른 접근 방식(Homography 변환, Bird's eye view 등) 검토 필요한가?

## 이 단계의 산출물

- Phase 3 Go/No-Go 판단 결과 문서
- Conditional인 경우: 원인 분석 + 해결 계획
- Go인 경우: Phase 3 세부 작업 목록
