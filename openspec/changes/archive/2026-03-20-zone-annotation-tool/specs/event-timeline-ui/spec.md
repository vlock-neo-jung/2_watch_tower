## ADDED Requirements

### Requirement: 침입 이벤트 마킹

시스템은 [침입 시작] / [침입 끝] 버튼으로 이벤트 구간을 마킹할 수 있어야 한다 (SHALL).

#### Scenario: 침입 시작 마킹
- **WHEN** zone이 선택된 상태에서 [침입 시작] 버튼을 클릭하면
- **THEN** 현재 프레임 번호가 시작 프레임으로 기록되고 "기록 중" 상태가 표시된다

#### Scenario: 침입 끝 마킹
- **WHEN** 기록 중 상태에서 [침입 끝] 버튼을 클릭하면
- **THEN** 현재 프레임이 끝 프레임으로 기록되고 이벤트가 목록에 추가된다

#### Scenario: 마킹 중 시작 재입력
- **WHEN** 기록 중 상태에서 [침입 시작]을 다시 클릭하면
- **THEN** 이전 마킹이 취소되고 현재 프레임이 새 시작점이 된다

#### Scenario: zone 미선택 시 마킹
- **WHEN** zone이 선택되지 않은 상태에서 [침입 시작]을 클릭하면
- **THEN** zone이 1개면 자동 선택되고, 2개 이상이면 "zone을 선택하세요" 안내가 표시된다

### Requirement: 이벤트 목록

기록된 이벤트를 목록으로 표시하고 관리할 수 있어야 한다 (SHALL).

#### Scenario: 이벤트 목록 표시
- **WHEN** 이벤트가 기록되면
- **THEN** zone 이름, 시작 시간, 끝 시간, 길이가 하단 목록에 표시된다

#### Scenario: 이벤트 클릭으로 이동
- **WHEN** 이벤트 목록에서 이벤트를 클릭하면
- **THEN** 영상이 해당 이벤트의 시작 프레임으로 이동한다

#### Scenario: 이벤트 삭제
- **WHEN** 이벤트의 [삭제] 버튼을 클릭하면
- **THEN** 해당 이벤트가 목록과 타임라인에서 제거된다

### Requirement: 타임라인 시각화

시간 슬라이더 위에 이벤트 구간을 색상 바로 표시해야 한다 (SHALL).

#### Scenario: 이벤트 구간 표시
- **WHEN** 이벤트가 기록되면
- **THEN** 시간 슬라이더 위에 해당 구간이 zone 색상으로 표시된다
