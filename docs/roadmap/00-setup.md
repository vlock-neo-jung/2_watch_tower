# Watch Tower - 1차 로드맵

**작성일:** 2026-03-17
**범위:** 서비스를 제공할 수 있는 기반 모델과 시스템 구축 (서비스화는 다음 단계)
**Linear:** [WatchTower 프로젝트](https://linear.app/fwy1216/project/watchtower-f6ee3203fb62/overview)

---

## 개요

건설 현장 AI 안전 모니터링의 핵심 기능(사람 검출, 추적, 위험구역 침입, 인원 탐지, PPE 감지)을 검증하고, 실시간 파이프라인으로 통합하여 서비스화 가능 여부를 판단한다.

### 전제 조건

- 1인 개발
- 개발 환경: WSL2 / Ubuntu 22.04 / RTX 3080 Ti (12GB) / CUDA 12.x
- 핵심 도구: uv, ultralytics, supervision, huggingface_hub
- 신규 모델 개발이 아닌 **공개 모델 선정 + 최소 튜닝 + 검증** 방식
- 과하지 않게, 점진적으로 확장 가능한 구조

### 접근 방식

구조화된 검증 환경(접근 B)을 채택한다:
- Python src layout + uv 의존성 관리
- YAML 설정 분리, 실험 결과 디렉토리 구조화
- 외부 MLOps 도구(DVC, W&B 등) 없이 시작, 필요 시 점진적 도입
- Docker 컨테이너화는 현 단계에서 제외 (멀티 환경/팀원 합류 시 도입)

## Phase 2: 핵심 기반 검증 (Person Detection + Tracking) ✅

**목표:** 모든 기능의 공통 기반인 "사람을 찾고 추적하는 것"이 현장 수준에서 동작하는지 검증
**목표일:** 2026-04-04 (1-2주)
**상태:** 완료 — 검증 리포트 00~05 참조. hazard 모델이 건설현장 최선, SODA-ft는 부적합 확인.

| 순서 | 작업 | 완료 기준 |
|------|------|----------|
| 1 | 검증용 데이터셋 확보 | SODA(사람 검출), MOT20(추적) 다운로드 및 정리 |
| 2 | Person Detection baseline | YOLO11m으로 SODA 평가, mAP 측정 |
| 3 | Tracking 적용 | ByteTrack 연동, 추적 안정성 확인 |
| 4 | 결과 분석 | 실패 케이스 정리, 거리/조건별 성능 분석 |
| 5 | Go/No-Go 판단 | 현장 적용 가능 여부 결론 |

**핵심 도구:** ultralytics, supervision, FiftyOne (결과 분석)

## Phase 3: 규칙 기반 기능 검증 (위험구역 침입 + 인원 탐지) ✅

**목표:** Phase 2의 Detection + Tracking 위에 규칙 로직을 올려서 기능 성립 여부 확인
**목표일:** 2026-04-18 (1-2주)
**상태:** 완료 — `src/watch_tower/zone/` 모듈 구현 (NEO-5). PolygonZone 기반 침입 감지 + 인원 카운팅 동작 확인.

| 순서 | 작업 | 완료 기준 |
|------|------|----------|
| 1 | Zone 규칙 구현 | supervision PolygonZone/LineZone 활용 |
| 2 | 위험구역 침입 감지 검증 | 사람 진입 시 이벤트 생성 확인 |
| 3 | 구역 내 인원 탐지 검증 | 실시간 인원 카운팅 정확도 측정 |
| 4 | 영상 기반 통합 테스트 | 테스트 영상으로 end-to-end 동작 확인 |
| 5 | 실패 케이스 및 한계 정리 | 오탐/미탐 분석, 조건별 한계 문서화 |

**핵심 도구:** supervision (zone/counting), OpenCV

## Phase 4: PPE 감지 검증 🔄

**목표:** 별도 모델(또는 동일 모델)로 PPE 감지 성능 확인, 최소 튜닝 적용
**목표일:** 2026-05-02 (1-2주)
**상태:** 진행 중 — Zone×PPE 결합 로직 프로토타입 구현 (NEO-6). NO-Hardhat 탐지 한계 확인, 이중 판정 방식 채택. 리포트 06 참조.

| 순서 | 작업 | 완료 기준 |
|------|------|----------|
| 1 | PPE 데이터셋 확보 | Construction-PPE, Mendeley PPE 다운로드 |
| 2 | PPE baseline 평가 | YOLO11m 11클래스로 mAP 측정 |
| 3 | 구역별 PPE 규칙 프로토타입 | "이 구역은 안전모+조끼 필수" 로직 |
| 4 | 최소 파인튜닝 (선택) | 현장 데이터 또는 공개 데이터로 정확도 개선 |
| 5 | Go/No-Go 판단 | 기능별 검증 결과 종합, 한계 정리 |

**핵심 도구:** ultralytics (학습/평가), CVAT (라벨링, 필요 시)
