# Object Detection 평가 수치 가이드

## 기본 개념

모델이 사진에서 객체를 찾으면, 정답(Ground Truth)과 비교하여 3가지 경우가 생긴다:

| 약어 | 이름 | 의미 |
|------|------|------|
| TP | True Positive | 실제 있는 걸 정확히 찾음 |
| FP | False Positive | 없는데 있다고 함 (오탐) |
| FN | False Negative | 있는데 못 찾음 (누락) |

## 핵심 수치

### Precision (정밀도)

"AI가 찾았다고 한 것 중 실제로 맞은 비율"

```
Precision = TP / (TP + FP)
```

- 10명을 찾았는데 8명이 진짜 사람 → 80%
- 높을수록 오탐이 적다

### Recall (재현율)

"실제 있는 것 중 AI가 찾아낸 비율"

```
Recall = TP / (TP + FN)
```

- 실제 10명인데 8명을 찾음 → 80%
- 높을수록 누락이 적다
- 안전 모니터링에서 가장 중요 (사람을 놓치면 위험)

### IoU (Intersection over Union)

AI가 그린 박스와 정답 박스의 겹침 비율. "맞춤" 판정 기준으로 사용된다.

```
IoU = 겹치는 영역 / 합친 영역
```

- `@50` → IoU 50% 이상이면 정답 인정 (느슨한 기준)
- `@50:95` → IoU 50%~95%를 5% 간격으로 평균 (엄격한 기준)

### mAP (mean Average Precision)

다양한 신뢰도 기준에서 Precision-Recall 균형을 종합한 점수.

| 표기 | 의미 | 용도 |
|------|------|------|
| mAP@50 | IoU 50% 기준 종합 점수 | 일반적 탐지 성능 평가 |
| mAP@50:95 | IoU 50~95% 평균 | 위치 정확도까지 포함한 엄격한 평가 |

## WatchTower Phase 2 Go/No-Go 기준

| 지표 | 기준 | 이유 |
|------|------|------|
| mAP@50 (Person) | >= 75% | PPE 목표(80%)의 기반, leakage 감안 보수적 설정 |
| Recall (Person) | >= 80% | 안전 모니터링에서 사람 누락은 치명적 |
