# SODA 데이터셋 기반 Person Detection 벤치마크

- 작성일: 2026-03-18
- 프로젝트: WatchTower (건설 현장 AI 안전 모니터링)
- 관련 이슈: FWY-53, FWY-51

## 1. 목적

이전 벤치마크(Grounding DINO 정답, 26장/96명)의 통계적 한계를 해소하고,
건설현장 특화 대규모 데이터셋으로 모델별 Person Detection 성능을 확정한다.

## 2. 데이터셋: SODA (Site Object Detection dAtaset)

| 항목 | 내용 |
|------|------|
| 출처 | BIM-LAB, South China University of Technology |
| 총 이미지 | 19,846장 (train 17,861 / test 1,985) |
| 어노테이션 | 286,201개 (VOC XML) |
| 해상도 | 1920x1080 (86%) |
| 클래스 | 15개 (person, helmet, vest, wood, board 등) |
| 벤치마크 사용 | test set 1,984장, Person 7,399개 |

### 이 데이터셋이 적합한 이유

- 건설현장 특화 — 실제 배포 환경과 동일 도메인
- 사람이 직접 라벨링 — Grounding DINO 자동 생성보다 정확
- 다양한 각도/거리/환경 포함
- 3개 모델 모두 SODA로 학습하지 않았으므로 독립 검증 보장

## 3. 벤치마크 결과

### SODA test set (1,984장, 7,399명, IoU≥0.5)

| 모델 | TP | FP | FN | Precision | Recall | F1 |
|------|-----|-----|------|-----------|--------|------|
| **COCO (0.25)** | 2,712 | 540 | 4,687 | 83.4% | **36.7%** | **50.9%** |
| hazard (0.15) | 2,643 | 475 | 4,756 | 84.8% | 35.7% | 50.3% |
| hazard (0.25) | 2,444 | 286 | 4,955 | **89.5%** | 33.0% | 48.3% |
| VisDrone (0.25) | 586 | 1,293 | 6,813 | 31.2% | 7.9% | 12.6% |
| VisDrone (0.15) | 698 | 3,108 | 6,701 | 18.3% | 9.4% | 12.5% |
| VisDrone (0.10) | 796 | 5,453 | 6,603 | 12.7% | 10.8% | 11.7% |

### 이전 벤치마크(Grounding DINO, 26장)와 비교

| 모델 | 소규모 Recall (26장) | 대규모 Recall (1,984장) | 차이 |
|------|-------|-------|------|
| COCO (0.25) | 43.8% | 36.7% | -7.1%p |
| hazard (0.25) | 30.2% | 33.0% | +2.8%p |
| VisDrone (0.25) | 9.4% | 7.9% | -1.5%p |

**순위 동일, 수치도 유사** — 소규모 벤치마크의 결론이 대규모에서도 유지됨.

## 4. 분석

### 4-1. 모델별 특성

**COCO (yolo11m.pt)**
- Recall 36.7%로 최고이지만 여전히 **63%를 놓침**
- Precision 83.4%로 높음 — 찾으면 대부분 정확
- COCO의 다양한 시점 학습이 건설현장에서도 일부 효과

**hazard (yolo11m-construction-hazard.pt)**
- COCO와 근소 차이 (Recall 33~35.7%)
- Precision이 가장 높음 (89.5%) — 건설현장 도메인 특화 효과
- conf=0.15로 낮추면 Recall +2.7%p 개선 가능

**VisDrone (yolov8m-visdrone.pt)**
- Recall 7.9~10.8%로 최악
- Precision도 12~31%로 낮음 — 오탐 다수
- 드론 시점 ≠ 고정 CCTV 시점. 학습 도메인 불일치

### 4-2. 공통 한계

모든 모델이 Recall 37% 미만 — **Person 7,399명 중 최대 2,712명만 찾음**.

이는 단순 모델 교체로 해결할 수 없는 수준이며, 다음 중 하나 이상의 조치가 필요:
1. 건설현장 고각 데이터로 파인튜닝
2. SAHI + COCO 조합 (아직 미테스트)
3. P2 탐지 헤드 등 아키텍처 수정
4. 카메라 각도/위치 제한

### 4-3. 이전 판단 재확인

| 시점 | 판단 | SODA 검증 |
|------|------|----------|
| P2-2 정성 평가 | 높은 각도 CCTV에서 감지 실패 | ✅ 확인 (Recall 33~37%) |
| FWY-52 COCO 교체 | COCO도 효과 없다 | ⚠️ 수정 — COCO가 최고 (hazard 대비 +3.7%p) |
| 소형객체 개선 | SAHI 효과 없음 | ✅ 확인 (hazard 모델 기준) |
| 소형객체 개선 | VisDrone 개선 | ❌ 완전 기각 (대규모 검증에서 최악) |

## 5. 다음 단계

### 즉시 실행

- **SAHI + COCO 벤치마크** — SAHI를 hazard 모델에만 테스트했었음. COCO가 최고인 걸 확인했으니 COCO+SAHI 조합을 SODA에서 정량 테스트

### 중기

- **COCO 기반 파인튜닝** — SODA train set(17,861장)으로 COCO 모델을 파인튜닝하면 건설현장 도메인 적응 + 고각 시점 학습 가능. 데이터가 이미 확보되어 있음

### 파인튜닝 시 기대 효과

현재 COCO Recall 36.7%에서, SODA 17,861장으로 파인튜닝하면:
- 건설현장 시점/배경에 특화
- Person 7,399개 GT로 높은 각도 사람 패턴 학습
- Recall 60~70% 이상 기대 가능 (도메인 적응 효과)
