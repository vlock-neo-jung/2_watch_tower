# SODA 파인튜닝 벤치마크

- 작성일: 2026-03-20
- 프로젝트: WatchTower (건설 현장 AI 안전 모니터링)
- 관련 이슈: FWY-61

## 1. 목적

SAHI + COCO 조합(Recall 57.8%)으로도 42%를 놓치는 한계가 확인되었다.
COCO pretrained yolo11m을 SODA train set(17,859장)으로 파인튜닝하여 고각 CCTV 시점의 Person Detection Recall을 개선한다.

## 2. 학습 설정

| 항목 | 설정 |
|------|------|
| Base 모델 | yolo11m.pt (COCO pretrained) |
| 학습 데이터 | SODA train set 17,859장 (63,447 Person bbox) |
| 평가 데이터 | SODA test set 1,984장 (7,399 Person bbox) |
| 이미지 크기 | 1280x1280 |
| 학습 전략 | 2단계 (Stage 1 → Stage 2) |

### 2단계 학습 전략

| 단계 | 설정 | Epochs |
|------|------|--------|
| Stage 1 | backbone freeze=10, 헤드만 학습 | 50 |
| Stage 2 | 전체 학습, lr0=0.001, cos_lr, patience=50 | 200 (early stop → 162) |

### 최종 학습 수치 (SODA val set 자체 평가)

| 지표 | 값 |
|------|-----|
| mAP50 | ~92.3% |
| mAP50-95 | ~52.4% |
| 종료 | epoch 162에서 early stopping (patience=50) |


## 5. 분석

### 5-1. SODA test set에서는 압도적 개선

SODA-ft (conf=0.25)는 SAHI 없이 단독으로도:
- Recall 84.1% — FWY-61 목표(60% 이상)를 크게 초과
- Precision 86.2% — COCO baseline보다 오히려 높음
- SAHI + COCO 대비 Recall +26.3%p, Precision +11.9%p 동시 개선

conf=0.15로 낮추면 Recall 87.5%까지 올라가지만 Precision이 79.2%로 하락한다.
conf=0.25가 Precision-Recall 균형이 가장 좋다.

### 5-1b. SAHI + SODA-ft 조합

SAHI를 추가하면:
- Recall: 84.1% → **90.2%** (+6.1%p) — 7,399명 중 6,672명 탐지, 놓치는 사람 727명
- Precision: 86.2% → 80.9% (-5.3%p) — SAHI 타일 경계 오탐 증가
- F1: 85.1% → 85.3% — Recall↑과 Precision↓이 상쇄되어 거의 동일

SAHI는 Recall을 추가로 올려주지만 Precision 하락 폭도 있다.
Recall 최대화가 목표라면 SAHI + SODA-ft, Precision-Recall 균형이면 SODA-ft 단독이 적합하다.

### 5-2. GT 26장에서는 오히려 하락 — 도메인 차이

| 비교 항목 | SODA 데이터 | GT 26장 |
|----------|------------|---------|
| 출처 | 학술 데이터셋 (도로/거리) | 실제 건설현장 CCTV |
| 시점 | 고각 감시카메라 (균일) | 헬멧캠, 고정CCTV, 타워크레인 등 (혼합) |
| 해상도 | 1920x1080 (균일) | 586x480 ~ 1280x720 (혼합) |
| 배경 | 도로, 보도, 교차로 | 건설 골조, 지하터널, 타워크레인 |

파인튜닝으로 SODA 도메인에 특화되면서 COCO가 갖고 있던 범용성을 잃었다.
Precision 100%는 "찾으면 정확하지만, 대부분을 못 찾는다"는 의미다.

### 5-3. 핵심 교훈

**파인튜닝은 학습 데이터의 도메인에서만 효과가 있다.**

- SODA 데이터 = 도로/거리 고각 카메라 → SODA 도메인에서 Recall 84.1%
- 실제 타겟 = 건설현장 CCTV → 건설현장에서는 Recall 19.8%로 하락
- 실 서비스 적용을 위해서는 **건설현장 데이터로 파인튜닝**하거나, **COCO + SAHI를 유지**해야 한다

## 6. Go/No-Go 판단

| 기준 | 판정 |
|------|------|
| FWY-61 목표 (SODA test Recall 60%+) | **Go** — 84.1% 달성 |
| 실 서비스 적용 가능성 | **No-Go** — 건설현장 도메인에서 성능 하락 확인 |

**결론:** SODA 파인튜닝 기법 자체는 검증되었다. 동일 전략을 건설현장 데이터에 적용하면 타겟 도메인에서도 유사한 개선을 기대할 수 있다.
