# 소형 객체 탐지 개선 시도 종합 리포트

- 작성일: 2026-03-18
- 프로젝트: WatchTower (건설 현장 AI 안전 모니터링)
- 관련 이슈: FWY-53, FWY-54, FWY-55, FWY-57

## 1. 배경

Phase 2 검증(리포트 00)에서 **높은 각도 CCTV에서 Person 감지 실패**가 확인되었다.
yolo11m-construction-hazard, yolo11m(COCO) 두 모델 모두 동일하게 실패.
근본 원인은 모델 선택이 아닌 **소형 객체 탐지의 한계**로 판단하고, 리서치 기반 개선 방안을 순차 테스트하였다.

### 제한사항

- 1인 개발자 (복잡한 아키텍처 수정 부담)
- 대량 데이터셋 자체 구축 불가 (구매는 가능)
- 완전 솔루션 구매 불가 (API 사용은 가능)

## 3. 시도한 방안 및 결과

### 3-1. FWY-54: SAHI (Slicing Aided Hyper Inference)

**가설**: 이미지를 640x640 타일로 분할하여 추론하면 소형 객체가 타일 내에서 상대적으로 커져 감지 가능.

**설정**: sahi 0.11.36, slice=640, overlap=0.2, perform_standard_pred=True

| 프레임 | SAHI OFF | SAHI ON | 변화 |
|--------|----------|---------|------|
| subway_1 | 1P | 1P | 0 |
| subway_2900 | 1P | 1P | 0 |
| crane_5700 | 4P | 4P | 0 |
| site_800 | 2P | 2P | 0 |

**결과: 효과 없음.** Person 감지 개선 전혀 없음.

**원인 분석**: SAHI가 해상도를 보존해도, 모델이 높은 각도에서의 사람 외형 패턴 자체를 학습하지 못했으므로 타일 안에서도 "사람"으로 인식하지 못함. 해상도 문제가 아니라 **학습 데이터의 시점(viewpoint) 문제**임이 확정.

### 3-2. FWY-55: 고해상도 입력 (imgsz=1280)

**가설**: YOLO 입력 크기를 640→1280으로 키우면 소형 객체 디테일이 4배 보존되어 감지 개선.

| 프레임 | imgsz=640 | imgsz=1280 | 변화 |
|--------|-----------|------------|------|
| subway_1 | 1P | 1P | 0 |
| subway_2900 | 1P | 1P | 0 |
| crane_5700 | 4P | 3P | -1 |
| site_800 | 2P | 1P | -1 |

**결과: 오히려 악화.** 리서치 경고 확인 — "640으로 학습된 모델을 1280으로 추론하면 feature map 분포가 달라져 성능 저하."

### 3-3. FWY-57: VisDrone 사전학습 모델 활용

**가설**: 드론/항공뷰 데이터(VisDrone)로 학습된 모델은 높은 각도에서의 사람 패턴을 알고 있으므로 감지 가능.

**모델**: mshamrai/yolov8m-visdrone (Hugging Face, VisDrone 데이터셋으로 학습된 YOLOv8m)

**클래스**: pedestrian(0), people(1), bicycle, car, van, truck, tricycle, awning-tricycle, bus, motor

| 프레임 | hazard 0.25 | COCO 0.25 | VisDrone 0.25 | VisDrone 0.15 | VisDrone 0.10 |
|--------|-------------|-----------|---------------|---------------|---------------|
| subway_1 | 1P | 1P | 0P | 1P | **3P** |
| subway_2900 | 1P | 0P | 2P | **4P** | **4P** |
| crane_5700 | 4P | 0P | 4P | **6P** | **6P** |
| site_800 | 2P | 0P | 0P | 0P | 0P |

**결과: 상당한 개선.**

- subway: 1P → 3~4P (3~4배 향상)
- crane: 4P → 6P (50% 향상)
- site: 여전히 0P (VisDrone 도메인과 다른 특성)

## 5. 핵심 결론

### 확정된 사실

1. **추론 단계 최적화(SAHI, 고해상도)로는 해결 불가** — 문제는 해상도가 아니라 학습 데이터
2. **높은 각도 데이터로 학습된 모델이 필요** — VisDrone 모델이 이를 증명
3. **VisDrone 모델은 높은 각도 CCTV에서 Person을 3~4배 더 잘 찾음**
4. **영상 특성에 따라 결과 차이 존재** — site_cctv(586x480, 중간 각도)에서는 VisDrone도 실패

### 남은 과제

1. **VisDrone 모델 + hazard 모델 조합 검토** — VisDrone은 Person 감지, hazard는 PPE 감지. 2단계 파이프라인 가능성
2. **VisDrone 모델로 전체 영상 Tracking 테스트** — 프레임 단위 테스트를 넘어 영상 전체에서 Tracking 안정성 확인
3. **site_cctv 실패 원인 분석** — 저해상도(586x480)? 다른 각도 특성?
4. **고객 현장 데이터 확보** — 실제 배포 환경에서의 최종 검증
