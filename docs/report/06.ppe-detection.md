# PPE 감지 검증 및 Zone×PPE 결합 로직

- 작성일: 2026-03-21
- 프로젝트: WatchTower (건설 현장 AI 안전 모니터링)
- 관련 이슈: NEO-6

## 1. 목적

`yolo11m-construction-hazard` 모델의 PPE 감지 성능을 Construction-PPE 데이터셋으로 공식 측정하고,
Zone 침입 감지([NEO-5](NEO-5: Zone 침입 감지))와 결합한 "Zone 내 PPE 규칙 위반" 프로토타입을 구현한다.

## 2. 평가 설정

| 항목 | 설정 |
|------|------|
| 모델 | `yolo11m-construction-hazard.pt` (11 classes) |
| 평가 데이터 | Construction-PPE val set (143장, 656 GT bbox) |
| 라벨 변환 | `construction-ppe-remapped` (클래스 인덱스 재매핑) |
| conf | 0.25 |
| IoU | 0.5 |
| 평가 기준 | mAP@50, mAP@50-95, Precision, Recall (per class) |

### 클래스 매핑

Construction-PPE 데이터셋은 11 classes를 정의하지만, 모델과 직접 대응되는 클래스는 4개다.

| 데이터셋 클래스 | 모델 클래스 (yolo11m-hazard) | 포함 여부 |
|----------------|-------------------------------|----------|
| helmet (0) | Hardhat (0) | ✅ |
| vest (2) | Safety Vest (7) | ✅ |
| Person (6) | Person (5) | ✅ |
| no_helmet (7) | NO-Hardhat (2) | ✅ |
| gloves (1), boots (3), goggles (4) | (매핑 없음) | ❌ 제외 |

## 4. 분석

### 4.1 강점

- **Person 탐지**: AP50=0.780, Recall=0.757 — 안정적. Zone 침입 감지의 기반으로 충분
- **Safety Vest**: AP50=0.789, Precision=0.913 — 조끼 착용 확인은 신뢰도 높음
- **Hardhat**: AP50=0.653, Precision=0.727 — 중간 수준, 안전모 착용 확인에 실용적

### 4.2 치명적 약점: NO-Hardhat 탐지 실패

| 지표 | 값 | 의미 |
|------|-----|------|
| NO-Hardhat AP50 | 0.028 | 사실상 감지 불가 수준 |
| NO-Hardhat Recall | 0.067 | 미착용 작업자 93%를 놓침 |

**원인 추정:**
- GT 불균형: 45개(NO-Hardhat) vs 201개(Hardhat) — 6.7배 차이
- "미착용" 클래스는 시각적으로 명확한 경계가 없어 학습 난이도 높음
- 안전모를 쓴 영상이 대부분인 데이터셋 특성

**영향:**
- 모델이 NO-Hardhat을 직접 탐지하는 방식은 신뢰 불가
- NO-Safety Vest 클래스도 GT가 없어 평가 불가 (동일 문제 예상)

### 4.3 오탐/미탐 패턴

| 상황 | 문제 |
|------|------|
| 헬멧 색상 다양 (흰/노랑/빨강) | 일부 색상 조합에서 미탐 증가 예상 |
| 안전모+사람 bbox 중첩 | person 내 hardhat 귀속 정확도에 영향 |
| 소형 작업자 (원거리) | SAHI 없이는 bbox 품질 저하 |

## 5. Zone×PPE 결합 로직 구현

### 5.1 설계 원칙

1. **Zone 우선**: ZoneProcessor가 침입 감지 → inside_tracker_ids 제공
2. **PPE 검사**: PPEViolationProcessor가 zone 내 작업자에 대해 PPE 착용 여부 판정
3. **이중 판정**: NO-Hardhat 직접 탐지(confidence=1.0) + Hardhat 미탐지(confidence=0.5)

### 5.2 PPE 모듈 구조

```
src/watch_tower/ppe/
├── __init__.py
├── models.py   # ZonePPERule, PPEViolationEvent, PPEClass
└── logic.py    # PPEViolationProcessor
```

### 5.3 PPE 귀속 로직 (PPE-Person 매핑)

PPE bbox 중심점이 Person bbox 내부에 있으면 해당 작업자에 귀속.

```
Person bbox: [x1, y1, x2, y2]
PPE center (cx, cy):
  x1 ≤ cx ≤ x2 AND y1 ≤ cy ≤ y2 → 이 작업자에 귀속
```

### 5.4 위반 판정 규칙

```python
for required_ppe in zone_rule.required_ppe:
    if NO-{required_ppe} detected near person:
        → 위반 (confidence=1.0)  # 명시적 미착용 탐지
    elif required_ppe NOT detected near person:
        → 위반 (confidence=0.5)  # 착용 클래스 미탐지
```

### 5.5 Zone별 PPE 규칙 예시

```python
# DANGER Zone (위험구역): 안전모 + 조끼 필수
ZonePPERule(
    zone_id="danger_zone",
    required_ppe=(PPEClass.HARDHAT, PPEClass.SAFETY_VEST),
)

# ENTRY Zone (출입구): 안전모만 필수
ZonePPERule(
    zone_id="entry_zone",
    required_ppe=(PPEClass.HARDHAT,),
)
```

### 5.6 통합 사용 예시

```python
# Zone 침입 감지 (기존 NEO-5)
zone_events = zone_processor.process(detections, fps=fps)
inside_tids = zone_processor.inside_tracker_ids

# PPE 위반 감지 (신규)
ppe_violations = ppe_processor.check_violations(
    detections=detections,
    inside_tracker_ids=inside_tids,
    fps=fps,
)

for v in ppe_violations:
    print(f"[PPE 위반] zone={v.zone_id}, tracker={v.tracker_id}, "
          f"미착용={v.missing_ppe}, confidence={v.confidence}")
```

## 7. Go/No-Go 판단

### PPE 탐지 모델

| 용도 | 판단 | 근거 |
|------|------|------|
| **Person + Hardhat + Safety Vest 탐지** | ✅ Go | AP50 0.65~0.79, 실용적 수준 |
| **NO-Hardhat 직접 탐지** | ❌ No-Go | AP50=0.028, 신뢰 불가 |
| **Zone×PPE 결합 (추정 방식 0.5)** | 🟡 조건부 Go | Recall 낮아 미탐 많음; 알림 임계값 조정 필요 |

### 한계 및 권고

1. **confidence 필터링**: 실제 알림은 `confidence=1.0` (명시적 탐지)만 사용하거나,
   confidence=0.5는 로그에만 기록하는 방식 권장
2. **NO-Hardhat 한계**: 현재 모델로 "안전모 미착용" 명시 탐지는 불가능에 가까움
   → "안전모 탐지 실패" 방식(confidence=0.5)이 실질적 대안
3. **PPE 전용 파인튜닝 필요**: NO-Hardhat AP50 0.5 이상 달성을 위한 PPE 특화 데이터 증강 및 재학습 권고
4. **SAHI 적용**: 원거리 소형 작업자에 대해 SAHI 조합 시 Recall 향상 기대
