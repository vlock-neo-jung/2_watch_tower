# SAHI + COCO 조합 벤치마크

- 작성일: 2026-03-18
- 프로젝트: WatchTower (건설 현장 AI 안전 모니터링)
- 관련 이슈: FWY-60, FWY-53

## 1. 목적

이전 SAHI 테스트(FWY-54)는 hazard 모델에만 적용하여 효과 없음으로 결론지었다.
SODA 벤치마크에서 COCO가 최고 모델로 확인되었으므로, SAHI + COCO 조합을 대규모 데이터로 정량 검증한다.

## 2. 테스트 설정

| 항목 | 설정 |
|------|------|
| 모델 | yolo11m.pt (COCO pretrained) |
| SAHI | slice=640x640, overlap=0.2, perform_standard_pred=True |
| 후처리 | GREEDYNMM, match_threshold=0.5 |
| 데이터 | SODA test set 1,984장, Person GT 7,399개 |
| 매칭 | IoU ≥ 0.5 |

## 3. 결과

| 모델 | TP | FP | FN | Precision | Recall | F1 |
|------|-----|-----|------|-----------|--------|------|
| **SAHI + COCO** | **4,275** | **1,475** | **3,124** | **74.3%** | **57.8%** | **65.0%** |
| COCO baseline | 2,712 | 540 | 4,687 | 83.4% | 36.7% | 50.9% |
| hazard (0.15) | 2,643 | 475 | 4,756 | 84.8% | 35.7% | 50.3% |
| hazard (0.25) | 2,444 | 286 | 4,955 | 89.5% | 33.0% | 48.3% |

### 개선 폭

```
Recall:    36.7% → 57.8%  (+21.1%p, 1.6배)
F1:        50.9% → 65.0%  (+14.1%p)
Precision: 83.4% → 74.3%  (-9.1%p)
```

## 4. 분석

### 4-1. 왜 SAHI + COCO는 효과가 있는가

이전에 hazard 모델에서 SAHI가 효과 없었던 이유가 명확해졌다:

```
hazard + SAHI = 효과 없음
  → hazard 모델은 높은 각도 사람 패턴 자체를 학습하지 못함
  → 타일로 잘라서 해상도를 보존해도 "이게 사람"을 모르면 무의미

COCO + SAHI = 대폭 개선
  → COCO는 다양한 시점의 사람 패턴을 학습
  → 작은 사람이 640x640 리사이징으로 뭉개지는 게 문제였음
  → SAHI로 해상도를 보존하면 COCO가 아는 패턴과 매칭됨
```

**결론: 문제는 "해상도 OR 패턴"이 아니라 "해상도 AND 패턴" 둘 다였다.**

### 4-2. Precision 하락 원인

Precision 83.4% → 74.3% (-9.1%p):
- SAHI가 타일별로 추론하므로 경계 영역에서 오탐 발생
- FP가 540 → 1,475로 증가 (2.7배)
- 하지만 TP가 2,712 → 4,275로 더 크게 증가 (1.6배)
- F1 기준으로는 명확한 개선

### 4-3. 여전히 42%를 놓침

Recall 57.8%는 이전 대비 대폭 개선이지만, 여전히 7,399명 중 3,124명을 놓치고 있다.
추가 개선 가능성:

| 방법 | 기대 효과 |
|------|----------|
| SODA 파인튜닝 (FWY-61) | 건설현장 시점 특화 학습 → Recall 추가 개선 |
| SAHI 파라미터 튜닝 | slice 크기, overlap 조정으로 소폭 개선 가능 |
| SAHI + 파인튜닝 모델 | 두 효과 결합 → 최대 Recall 기대 |
