# Watch Tower - 프로젝트 개요 및 요구사항 정의서

## 1. 프로젝트 개요

**프로젝트명:** Watch Tower

**한줄 요약:** 건설 현장 실시간 AI 안전 모니터링 시스템

**목적:** 건설 현장의 안전사고를 사전에 예방하기 위해, 카메라 영상을 실시간 AI로 분석하여 보호구 미착용, 작업자 쓰러짐, 위험구역 침입, 구역 내 인원 수를 자동 감지하고, 중앙에서 모니터링하며 즉각 알림을 발송하는 시스템

**핵심 방향:**

- 직접 개발을 최소화하고, 검증된 오픈소스 및 상용 구성요소를 조합하여 빠르게 구축
- 상용화 가능한 수준의 품질 확보
- 소규모(1~5대)에서 시작하여 중규모(10~50대) 이상으로 확장 가능한 구조

## 2. 핵심 기능 요구사항

### 2.1 영상 수집

- 웹캠(USB) 및 IP카메라(RTSP)를 통한 실시간 영상 수집
- 멀티 카메라 동시 입력 지원
- 해상도 및 프레임 레이트 조절 가능 (성능 최적화용)

### 2.2 PPE(개인보호구) 착용 감지

- **필수 감지 항목:** 안전모(Hard hat), 안전장갑(Safety gloves)
- **선택 감지 항목:** 안전조끼(Safety vest), 마스크(Face mask)
- 구역/공정별 필수 PPE 규칙 설정 가능 (예: 크레인 구역 = 안전모+조끼)
- 미착용 시 이벤트 생성 및 기록
- 장갑 등 소형 객체는 근접 카메라 배치 등 설치 설계로 보완

### 2.3 작업자 쓰러짐 감지

- 작업자의 쓰러짐(낙상/실신) 실시간 감지
- 정상 작업 자세(웅크림, 허리 굽힘 등)와 실제 쓰러짐 구분 (오탐 최소화)
- 감지 시 이벤트 생성 및 기록

### 2.4 위험구역 침입 감지

- 카메라 화면 기준으로 위험구역(Polygon ROI) 또는 가상선(Virtual Line) 설정 가능
- 작업자 또는 차량의 위험구역 진입 실시간 감지
- 구역별 활성 시간대, 체류시간, 감지 대상 등의 규칙 설정 가능
- 구역 진입 시 이벤트 생성 및 기록
- 구역별 필수 PPE 규칙과 연계 가능 (예: 크레인 작업구역 = 안전모+조끼 필수)

### 2.5 구역 내 인원 탐지

- 지정된 구역(Polygon ROI) 기준으로 현재 인원 수 실시간 집계
- 구역별 최대 동시 인원 수 기록 및 조회
- 구역별 허용 인원 초과 시 이벤트 생성 및 기록
- 입장/퇴장 방향 기반 인원 카운팅 확장 가능

### 2.6 중앙 모니터링 대시보드

- 멀티카메라 라이브 화면 동시 표시
- AI 감지 결과 실시간 오버레이 (바운딩 박스 등)
- 이벤트 타임라인 (위반 이력 시각화)
- 구역별 위반 통계 및 리포팅
- 구역별 현재 인원 수 및 최대 동시 인원 표시
- 웹 브라우저 기반 접근

### 2.7 실시간 알림

- 위험 감지 시 담당자에게 즉각 알림 전송
- 알림 채널: 모바일 Push, SMS, 카카오 알림톡 등
- 알림톡 수신 불가 시 SMS 자동 우회 발송 (Fallback)
- 미확인 시 재알림 등 에스컬레이션 규칙 지원
- 알림 메시지에서 관제 화면으로 바로 접근 가능

## 3. 비기능 요구사항

| 항목 | 요구사항 |
|------|---------|
| 실시간성 | 감지 → 알림 전달까지 수초 이내 |
| 정확도 | PPE 감지 mAP 80% 이상, 쓰러짐 감지 정확도 90% 이상 (목표) |
| 가용성 | 네트워크 단절 시 로컬 버퍼링 및 복구 후 재전송 |
| 확장성 | 카메라 1~5대(소규모) → 10~50대(중규모) 확장 가능 구조 |
| 보안 | 영상 데이터 암호화, 접근 권한 관리, 감사 로그 |

## 4. 제약 조건

- 기존에 존재하는 오픈소스 및 상용 라이브러리를 최대한 활용하여 개발 기간 단축
- 오픈소스 사용 시 상용화 가능한 라이선스(Apache 2.0 등) 우선 검토
- 개인정보보호법 등 관련 법규 준수 필수 (별도 문서로 정리 예정)
